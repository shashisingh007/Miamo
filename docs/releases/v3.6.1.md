## Documentation conventions (enforced)

1. **Pair-write tone** — non-technical paragraph first, technical second, for every concept.
2. **The four users** — Priya / Arjun / Karan / Riya, consistent across all 15 docs.
3. **Real numbers** — every weight, threshold, TTL cited from source code.
4. **Citations** — every algorithm reference cites a file path; every endpoint reference cites a server.ts handler.
5. **No marketing words** — banned: *leverage*, *synergy*, *best-in-class*, *next-generation*, *cutting-edge*.
6. **Mermaid for flows** — sequence diagrams + topology.
7. **Every new feature ships its doc** — PR with feature code but no doc update gets rejected.
